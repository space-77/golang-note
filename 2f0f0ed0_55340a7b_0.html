【1】代码入门： 

  1. package main
  3. import "fmt"
  5. //接口的定义：定义规则、定义规范，定义某种能力：
  6. type SayHello interface{
  7.         //声明没有实现的方法：
  8.         sayHello()
  9. }
  12. //接口的实现：定义一个结构体：
  13. //中国人：
  14. type Chinese struct{
  16. }
  18. //实现接口的方法---》具体的实现：
  19. func (person Chinese) sayHello(){
  20.         fmt.Println("你好")
  21. }
  23. //接口的实现：定义一个结构体：
  24. //美国人：
  25. type American struct{
  27. }
  28. //实现接口的方法---》具体的实现：
  29. func (person American) sayHello(){
  30.         fmt.Println("hi")
  31. }
  34. //定义一个函数：专门用来各国人打招呼的函数，接收具备SayHello接口的能力的变量：
  35. func greet(s SayHello){
  36.         s.sayHello()
  37. }
  40. func main(){
  41.         //创建一个中国人：
  42.         c := Chinese{}
  43.         //创建一个美国人：
  44.         a := American{}
  46.         //美国人打招呼：
  47.         greet(a)
  48.         //中国人打招呼：
  49.         greet(c)
  50. }


【2】总结：
（1）接口中可以定义一组方法，但不需要实现，不需要方法体。并且接口中不能包含任何变量。到某个自定义类型要使用的时候（实现接口的时候）,再根据具体情况把这些方法具体实现出来。
（2）实现接口要实现所有的方法才是实现。
（3）Golang中的接口不需要显式的实现接口。Golang中没有implement关键字。
（Golang中实现接口是基于方法的，不是基于接口的）
例如：
A接口 a,b方法
B接口 a,b方法
C结构体 实现了  a,b方法 ，那么C实现了A接口，也可以说实现了B接口   （只要实现全部方法即可，和实际接口耦合性很低，比Java松散得多）
（4）接口目的是为了定义规范，具体由别人来实现即可。
